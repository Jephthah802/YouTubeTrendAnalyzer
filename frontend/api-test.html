How to Fix Your API Issue

Your 404 error is caused by API routing issues. Follow these steps:

  1. Update your Express routes to remove the '/api' prefix since Vercel already routes '/api/*' to your function
  2. Fix your vercel.json configuration to properly route requests
  3. Test your endpoints using the buttons below to verify they work

Example fix: Change app.use('/api', apiRoutes) to app.use('/', apiRoutes)

Required Vercel Configuration

Create a vercel.json file in your project root with this content:

{
  "version": 2,
  "builds": [
    {
      "src": "api/index.js",
      "use": "@vercel/node"
    },
    {
      "src": "frontend/**/*.html",
      "use": "@vercel/static"
    }
  ],
  "routes": [
    {
      "src": "/api/(.*)",
      "dest": "/api/index.js"
    },
    {
      "src": "/",
      "dest": "/frontend/home.html"
    },
    {
      "src": "/(.*)",
      "dest": "/frontend/$1"
    }
  ]
}

Express Server Fix

Update your api/index.js to remove the '/api' prefix from routes:

// Remove the /api prefix from your route mounts
app.use('/', apiRoutes);          // instead of app.use('/api', apiRoutes);
app.use('/auth', authRoutes);     // instead of app.use('/api/auth', authRoutes);
app.use('/favorites', favoritesRoutes); // instead of app.use('/api/favorites', favoritesRoutes);
app.use('/playlists', playlistRoutes); // instead of app.use('/api/playlists', playlistRoutes);
